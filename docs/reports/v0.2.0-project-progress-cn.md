# AeroSpec Agent v0.2.0 项目进展报告

**报告日期**: 2026-05-26
**报告版本**: v0.2.0
**报告人**: zweien / Claude (Anthropic)

---

## 一、项目概述

AeroSpec Agent 是一款面向概念设计阶段的无人机气动布局智能设计平台。用户通过自然语言描述设计需求，系统基于大语言模型（LLM）解析生成结构化参数规格，调用 OpenVSP 几何引擎自动生成 CAD 模型，并在 Web 端提供三维预览、参数面板、版本管理和气动分析等功能。

v0.1.0（2026-05-22 发布）已验证了从自然语言到 CAD 模型的端到端流程，支持常规布局、双尾撑、飞翼和翼身融合 4 种气动布局。v0.2.0 在此基础上将布局覆盖范围扩展至 11 种，并显著提升了气动分析能力和设计变体策略的智能化水平。

---

## 二、v0.2.0 核心成果

### 2.1 气动布局大幅扩展（4 种 → 11 种）

本次 release 新增 7 种气动布局，覆盖范围从基础布局扩展到涵盖前翼、串列翼、双翼、连接翼、箱式翼和双机身等实验性构型：

- **鸭翼布局（canard）**: 在主翼前方增设鸭翼，适用于高机动需求场景
- **三翼面（three_surface）**: 鸭翼 + 主翼 + 尾翼的组合，提供更多的配平控制能力
- **串列翼（tandem_wing）**: 前后串列布置两片机翼，无传统尾翼
- **双翼机（biplane）**: 上下两层机翼叠加，带垂直间距
- **连接翼（joined_wing）**: 后翼前掠并与主翼在翼尖处连接
- **箱式翼（box_wing）**: 上下翼面 + 翼尖端板形成闭合升力面
- **双机身（multi_fuselage）**: 两个机身并排，中央共用主翼

每种布局均具备完整的 Schema 定义、OpenVSP 几何构建器、参数默认值、E2E 测试、前端二维预览和三维模型支持。

### 2.2 VSPAERO 多面气动分析

v0.1.0 仅分析主翼单一表面，v0.2.0 实现了**基于布局类型的多表面联合分析**：

- 鸭翼/三翼面布局：分析主翼 + 鸭翼
- 串列翼/连接翼：分析主翼 + 后翼
- 双翼机：分析上翼 + 下翼
- 箱式翼：分析主翼 + 下翼面

分析结果通过 `VspaeroReport.components_analyzed` 字段明确列出参与计算的表面组件，使分析过程更加透明。需要强调的是，VSPAERO 采用面板法（Panel Method），结果仅供概念阶段对比参考，不适用于工程决策。

### 2.3 Deep Design 布局感知变体策略

Deep Design 功能在 v0.1.0 中仅对 `wing.span` 进行统一调整。v0.2.0 引入 **LAYOUT_STRATEGIES** 策略字典，针对不同布局类型调整差异化参数：

| 布局类型 | 变体参数 | 调整范围 |
|---------|---------|---------|
| 鸭翼/三翼面 | 鸭翼翼展 | ±0.5m |
| 串列翼/连接翼 | 后翼翼展 | ±1m |
| 双翼机 | 上下翼间距 | ±0.2m |
| 箱式翼 | 箱式翼间距 | ±0.3m |
| 其他布局 | 主翼翼展 | ±2m |

这一改进使 Deep Design 生成的变体更加贴合各布局类型的结构特征，提升了设计探索的有效性。

### 2.4 前端增强

- **布局类型标签**: CAD 预览器头部新增布局类型标识（如"鸭翼布局"、"箱式翼"），便于用户快速识别当前设计
- **布局提取工具**: `previewGeometry.ts` 导出 `extractLayout()` 和 `LAYOUT_LABELS`，支持多组件复用

### 3.2 多维度 QA 验证体系

v0.2.0 建立了三层验证体系，明确区分软件管道可靠性和工程可信度：

1. **Pipeline QA**（`docs/layout-openvsp-qa.md`）: 验证软件管道结构完整性，11/11 通过
2. **Real OpenVSP QA**（`docs/layout-openvsp-real-qa.md`）: 验证真实 OpenVSP 3.50.2 几何生成，11/11 通过
3. **Visual QA**（`docs/layout-visual-qa.md`）: 人工视觉检查三维模型合理性，11/11 通过

此外，建立了**布局成熟度矩阵**（`docs/layout-maturity-matrix.md`），将"稳定"细分为五个维度：管道稳定、OpenVSP 验证、前端预览、VSPAERO 分析、Deep Design 兼容。所有 11 种布局在软件管道维度均达到验证标准，但**没有任何布局达到工程验证级别**。

## 四、已知限制

当前版本为概念设计原型工具，存在以下明确限制：

1. **非工程设计工具**: 几何生成可靠不代表气动性能最优或结构可行
2. **面板法近似**: VSPAERO 结果为近似值，不含机身/尾撑等非翼面部件
3. **LLM 生成风险**: 大模型可能遗漏布局特定参数，由 `spec_defaults.py` 自动填充兜底
4. **单用户原型**: 无多用户协作、持久化存储或权限管理
5. **固定翼限制**: 仅支持固定翼无人机，不支持旋翼、倾转旋翼或载人飞行器

## 五、下一步计划

### 近期（v0.2.x）

- **多视角视觉检查**: 为复杂布局（双尾撑、连接翼）补充侧视、顶视截图
- **材质区分**: OpenVSP 导出时赋予不同部件不同颜色，提升视觉辨识度
- **LLM 布局生成优化**: 验证大模型对各实验性布局的 spec 生成能力

### 中期（v0.3.0 方向）

- **工程可信度提升**: 引入 VLM（涡格法）或低阶 CFD 作为更高精度分析选项
- **设计规则增强**: 基于布局类型的气动稳定性初判（静稳定度估算）
- **参数敏感性分析**: 量化各参数对性能指标的敏感程度

### 长期

- **结构可行性初筛**: 基于几何参数估算翼梁载荷分布
- **多目标优化**: 将航程、载荷、速度等多目标纳入设计优化流程

---

## 六、总结

AeroSpec Agent v0.2.0 在 4 天内（v0.1.0 → v0.2.0）将气动布局支持从 4 种扩展到 11 种，建立了覆盖 Schema、几何生成、气动分析、前端预览和智能变体的完整能力。通过三层 QA 验证体系（管道/真实 OpenVSP/视觉），所有布局的软件管道可靠性均得到验证。

项目当前处于**概念设计工具**阶段，软件管道成熟但工程验证尚未开展。建议在后续版本中逐步引入更高精度的分析方法和设计规则校验，向工程辅助决策工具演进。

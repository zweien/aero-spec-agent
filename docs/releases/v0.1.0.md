# v0.1.0 — Concept Design Workbench MVP

**Release Date**: 2026-05-22

## Overview

AeroSpec Agent v0.1.0 is a natural-language aircraft concept design workbench MVP. Users describe an aircraft in plain language, the system generates an AircraftSpec, creates CAD models, and provides an interactive 3D preview with design exploration tools.

## Features

### Core Workflow
- **Chat-driven design generation**: Natural language → structured AircraftSpec via LLM
- **Dual CAD backend**: Fake CAD (deterministic placeholder) + OpenVSP 3.50.2 (real geometry)
- **Interactive 3D preview**: Three.js GLB/OBJ model viewer with part selection
- **Live parameter editing**: Inline value editing with source tracking (user/inferred/default)

### AI-Powered Design
- **Deep Design exploration**: LangGraph-based multi-variant design with configurable depth (2-5 variants)
- **Optimization strategies**: Endurance, speed, payload, STOL
- **AI recommendation**: Variant recommendation with reasoning
- **No-tool-call fallback**: Supports models without function calling (MiniMax-M2.5, etc.)

### Analysis & Comparison
- **Design rules validation**: 15+ rules covering dimensional, configuration, and loading checks
- **Performance estimation**: 3-layer heuristic (aerodynamic, weight, mission)
- **VSPAERO panel method**: Optional wing-only aerodynamic sweep
- **Compare View**: Side-by-side comparison of up to 5 designs with metrics table
- **Markdown export**: Full comparison report with confidence indicators
- **Variant trust system**: Confidence levels (high/medium/low) based on backend type, metrics source, and analysis availability

### Demo & Presentation
- **3 demo scenarios**: Long-endurance UAV, high-speed recon, heavy-lift cruiser
- **One-command demo seed**: `python scripts/seed_demo_designs.py`
- **LLM profile management**: Save/switch between API configurations

## Known Limitations

- Fake CAD produces placeholder geometry, not engineering models
- VSPAERO analysis limited to wing-only
- Performance estimates are heuristic-based, not CFD
- Job state is not persisted across server restarts
- Only conventional tail, under-wing/nose engines supported
- Fixed-wing UAV only (no rotorcraft, no blended wing body)

## System Requirements

- Python 3.11+
- Node.js 18+
- OpenAI-compatible LLM API (DeepSeek, OpenAI, etc.)
- OpenVSP 3.50.2 (optional, for real CAD generation)

## Quick Start

```bash
# Backend
python -m venv .venv && . .venv/bin/activate && pip install -e ".[dev]"
set -a && . ./.env && set +a
CAD_BACKEND=fake .venv/bin/python -m uvicorn services.api.app.main:app --host "$API_HOST" --port "$API_PORT"

# Frontend
cd apps/web && npm install && npm run dev

# Demo data
python scripts/seed_demo_designs.py
```

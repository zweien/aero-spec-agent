# v0.2.0 — Expanded Aerodynamic Layouts and Analysis

**Release Date**: 2026-05-26
**Since**: v0.1.0 (2026-05-22)
**Commits**: 62

## Overview

AeroSpec Agent v0.2.0 将支持的气动布局从 4 类扩展到 11 类，每种布局均有完整的 schema 定义、OpenVSP 几何构建器、spec 默认值、E2E 测试、前端预览、VSPAERO 多面分析和布局感知的 Deep Design 变体策略。

## Features

### 7 New Aerodynamic Layouts

新增 7 种布局类型，加上 v0.1.0 的 4 种，共 11 种：

| Layout | Chinese Label | Key Feature |
|--------|--------------|-------------|
| `conventional` | 常规布局 | Standard wing + tail |
| `twin_boom` | 双尾撑 | Twin boom structure |
| `flying_wing` | 飞翼 | No fuselage, no tail |
| `blended_wing_body` | 翼身融合 | BWB flat body |
| `canard` | 鸭翼布局 | Forward canard surface |
| `three_surface` | 三翼面 | Canard + main wing + tail |
| `tandem_wing` | 串列翼 | Two wings in tandem |
| `biplane` | 双翼机 | Stacked wings with gap |
| `joined_wing` | 连接翼 | Forward-swept rear wing |
| `box_wing` | 箱式翼 | Lower wing + endplates |
| `multi_fuselage` | 双机身 | Paired fuselages |

### VSPAERO Multi-Surface Analysis

- Layout-aware analysis: VSPAERO now includes all aerodynamic surfaces based on layout
- Canard layouts analyze canard + main wing; biplane analyzes upper + lower wing
- New `components_analyzed` field in `VspaeroReport` lists which surfaces were included
- Results reflect combined multi-surface aerodynamics

### Deep Design Layout-Aware Strategies

- Variant generation varies layout-specific fields, not just `wing.span`
- Canard: varies `canard.span` ±0.5m; Biplane: varies `second_wing.gap` ±0.2m
- Tandem/Joined: varies `rear_wing.span` ±1m; Box: varies `box_wing_config.gap` ±0.3m
- 11 layout-specific strategies via `LAYOUT_STRATEGIES` dictionary

### Layout QA Infrastructure

- `scripts/validate_layout_matrix.py` — batch validation for all 11 layouts
- Generates `docs/layout-openvsp-qa.md` with per-layout artifact verification
- Supports both fake and real OpenVSP backends
- YAML front matter compatible with `summarize_qa_status.py`

### Frontend Layout Badge

- CAD viewer header now shows layout type label (e.g., 常规布局, 鸭翼布局)
- Layout extraction utility exported for reuse

### Documentation

- `docs/layout-maturity-matrix.md` — per-layout maturity assessment across 10 dimensions
- All 11 layouts verified as **Stable**
- README updated with 654+ backend tests, multi-surface VSPAERO, layout-aware Deep Design

## Known Limitations

- **Concept design tool only** — layout support means parametric geometry generation, not engineering validation. "Pipeline Stable" and "OpenVSP Verified" mean the software pipeline works reliably; they do NOT mean the configuration is aerodynamically optimal, structurally feasible, or engineering certified.
- **VSPAERO panel method** — approximate results, not suitable for design decisions; non-wing bodies (fuselage, booms) not included in analysis
- **Visual inspection pending** — 3D models have not been systematically checked by humans for geometric plausibility. Complex layouts (canard, box_wing, multi_fuselage) especially need visual review.
- **LLM spec generation** — LLM may omit layout-specific fields; mitigated by `spec_defaults.py` auto-fill
- **No per-surface reports** — VSPAERO outputs combined aerodynamic metrics only
- **No CFD integration** — higher-fidelity analysis tools remain outside scope
- **Single-user prototype** — no multi-user or persistence features
- **Fixed-wing UAV only** — no rotorcraft, tiltrotor, or manned aircraft

## Upgrade Notes

No breaking changes from v0.1.0. All existing API endpoints, data formats, and frontend interfaces remain compatible.

- `VspaeroReport.components_analyzed` is a new optional field; frontend ignores if absent
- Deep Design strategy change is backend-only; frontend still shows compact/standard/extended labels
- `extractLayout()` and `LAYOUT_LABELS` are new exports from `previewGeometry.ts`

## Quick Start

```bash
# Backend
python -m venv .venv && . .venv/bin/activate && pip install -e ".[dev]"
set -a && . ./.env && set +a
CAD_BACKEND=fake .venv/bin/python -m uvicorn services.api.app.main:app --host "$API_HOST" --port "$API_PORT"

# Frontend
cd apps/web && npm install && npm run dev

# Validate all layouts
CAD_BACKEND=fake python scripts/validate_layout_matrix.py
```

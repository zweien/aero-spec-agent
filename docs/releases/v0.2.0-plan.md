## v0.2.0 Goals

### Must-Have (P0)

- [x] 11 类 `aircraft.layout` schema 定义及 YAML 示例
- [x] 每种布局有 OpenVSP builder，能生成 vsp3 + glb 产物
- [x] `spec_defaults.py` 覆盖所有布局的布局特有字段
- [x] OpenVSP E2E 测试覆盖全部 11 种布局
- [x] Fake CAD E2E 测试覆盖全部 11 种布局
- [x] 前端 2D SVG 预览正确渲染全部 11 种布局
- [x] Layout maturity matrix 文档
- [x] 11-layout QA 验证脚本 + 报告
- [x] VSPAERO 多面分析（layout-aware）
- [x] Deep Design 布局感知变体策略
- [x] Compare View 兼容所有布局
- [x] README 与代码能力一致
- [x] License / third-party notices 保持准确
- [ ] v0.2.0 release note 和 tag

### Should-Have (P1)

- [ ] LLM prompt / tool schema 稳定性验证（每种布局至少 1 次成功生成）
- [ ] UI 布局成熟度标识（前端显示 maturity badge）
- [ ] 复杂布局人工几何检查报告（canard, biplane, box_wing 等）

### Nice-to-Have (P2)

- [ ] 前端布局选择器改进（布局切换时显示推荐参数）
- [ ] VSPAERO 每面独立报告
- [ ] OpenVSP 产物截图嵌入 QA 报告

## Non-Goals

v0.2.0 **明确不做**：

- **CFD 级高保真气动分析** — VSPAERO panel method 是近似方法，不替代 CFD
- **工程定型判断** — 布局生成通过不等于工程可行，不做工程认证
- **多用户权限系统** — 单用户原型阶段
- **大型架构重构** — 在现有架构上扩展，不重新组织
- **商业级安装器** — pip install + npm install 手动安装
- **Rotorcraft / tiltrotor** — 仅固定翼 UAV
- **实时协同编辑** — 单用户顺序操作

## Risks

| Risk | Impact | Mitigation |
|------|--------|-----------|
| LLM 生成不完整的 layout-specific fields | 中 | `spec_defaults.py` 自动补齐；chatTools schema 列出所有字段 |
| OpenVSP 复杂布局几何需人工检查 | 中 | QA 报告标注 Recommendations；maturity matrix 标注 Risk |
| VSPAERO 对复杂构型精度有限 | 低 | 文档明确标注"panel method, not CFD"；结果仅供参考 |
| README 容易夸大工程成熟度 | 中 | 每处添加 disclaimer；maturity matrix 定义严格 |
| 许可证说明需要保持清晰 | 低 | THIRD_PARTY_NOTICES.md 已建立；每次添加依赖时更新 |
| LLM 对新布局字段不稳定 | 中 | 系统提示词包含所有布局定义；手动验证后标记 P1 |

## 阶段 2：真实 OpenVSP 全链路验收

### 2.1 新增脚本

**`scripts/validate_openvsp_pipeline.py`**

```python
def main():
    # 1. 检查 OpenVSP 可用（复用 check_openvsp_env 逻辑）
    # 2. 单发常规布局 UAV 验证
    spec_single = load_aircraft_spec("packages/aircraft-schema/examples/twin_engine_uav.yaml")
    spec_single.engine.count = 1  # 改为单发
    result = generate_aircraft(spec_single, tmp_path, backend=OpenVspBackend())
    validate_artifacts(result)
    
    # 3. 双发常规布局 UAV 验证
    spec_twin = load_aircraft_spec("packages/aircraft-schema/examples/twin_engine_uav.yaml")
    result = generate_aircraft(spec_twin, tmp_path, backend=OpenVspBackend())
    validate_artifacts(result)
    
    # 4. 输出 JSON 报告
```

### 2.2 Artifact 验证

每个生成结果检查：
- `.vsp3` 存在且 > 0 字节
- `.step` 存在且 > 0 字节（best-effort，STEP 可能导出失败）
- `.obj` 或 `.glb` 存在且有效
- `aircraft_spec.yaml` 存在
- `generation_log.json` 可解析且含 expected 字段
- `validation_report.json` 可解析且含 design_metrics

### 2.3 Graceful Skip

OpenVSP 不可用时：
- 输出 SKIP + 原因
- exit code 0
- 记录到 docs/openvsp-validation-report.md

## 阶段 3：Deep Design 可信度增强

### 3.1 后端数据结构

**新增 `VariantTrust` 数据类（services/api/app/schemas/ 或 services/workers/ 下）**：

```python
@dataclass
class VariantTrust:
    backend: str                        # "fake" | "openvsp"
    metrics_source: str                 # "backend_design_metrics" | "performance_estimate" | "client_heuristic"
    defaulted_parameter_count: int      # 默认补全参数数量
    warnings: list[str]                 # 警告列表
    confidence_level: str               # "high" | "medium" | "low" | "unknown"
    confidence_reasons: list[str]       # 置信度判定原因
    generated_by: str                   # "fake_cad" | "openvsp"
    has_real_geometry: bool             # 是否有真实几何（非 placeholder）
    has_aero_analysis: bool             # 是否有气动分析
```

### 3.2 置信度规则

```
confidence_level = "low" if:
    - backend == "fake" (Fake CAD 永远 low)
    - metrics_source == "client_heuristic"
    - defaulted_parameter_count >= 5

confidence_level = "medium" if:
    - backend == "openvsp"
    - metrics_source == "performance_estimate"
    - not has_aero_analysis

confidence_level = "high" only if:
    - backend == "openvsp"
    - metrics_source == "backend_design_metrics"
    - has_aero_analysis
    - defaulted_parameter_count < 3
```

### 3.3 计算位置

在 `generate_aircraft()` 末尾，基于已有的 generation result 数据计算 VariantTrust，写入 `validation_report.json` 的 `variant_trust` 字段。

### 3.4 前端显示

- 显示置信度标签（高/中/低）+ 颜色

**Markdown 报告**：

## 阶段 4：Compare View 工程化完善

### 4.1 指标分组重构

当前 METRIC_ROWS 已有 3 组（基础尺寸/性能估算/可信度与风险），保持不变但增加分组说明文案。

### 4.2 最佳值解释

**`apps/web/src/components/compare/bestValue.ts`** 修改：

```typescript
// 当前：返回 boolean
export function isBest(key, index, metricsMap): boolean

// 新增：返回 { isBest: boolean, reason: string }
export function bestValueExplanation(key, index, metricsMap): { isBest: boolean; reason: string }
```

每个指标有解释模板：
- wingspan_m: "翼展最大/最小，影响航程和机动性"
- estimated_range_km: "估算航程最远/最近"
- wing_loading_kg_m2: "翼载荷最低/最高，影响起降和机动性"
- risk_level: "风险等级最低"

### 4.3 推荐方案

新增 `apps/web/src/components/compare/recommendVariant.ts`：

```typescript
export function recommendVariant(items: CompareItem[]): {
  recommendedId: string | null;
  reason: string;
} | null
```

规则：
- 所有方案 confidence == "low" 时不推荐（返回 null）
- 否则基于综合评分（性能指标权重 + 风险权重）推荐
- 输出推荐原因（中文，1-2 句）

### 4.4 NaN/undefined/null 防御

**metricExtractors.ts** — 所有值提取后做防御性转换：
```typescript
const safeNum = (v: unknown): number | undefined =>
  (typeof v === "number" && Number.isFinite(v)) ? v : undefined;
```

缺失值统一显示 "未计算"（CompareMetricCell.tsx 已有 "暂无" 显示，统一文案）。

### 4.5 空状态和异常状态

- CompareDrawer：0 个 item 显示引导文案（已有），1 个 item 显示 "至少添加 2 个方案"
- 所有指标为空时显示 "暂无指标数据"
- CompareTable：items.length === 0 不渲染表格

### 4.6 Markdown 报告增强

exportCompareReport.ts 增加：
- 指标分组标题
- 推荐方案章节（如有）
- 最佳值解释
- 可信度说明（每个方案的 confidence + warnings）

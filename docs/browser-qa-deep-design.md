# Deep Design 浏览器 QA 清单

## 环境

- 前端：`http://localhost:3900`
- 后端：`http://localhost:8900`
- CAD_BACKEND=fake

---

## 工作流 QA

### W1: Chat → Deep Design → Set Current Variant 闭环

1. 打开 `http://localhost:3900`
2. 在对话面板输入："设计一架翼展10米、单发、上单翼的长航时无人机"
3. 等待生成完成 → CAD Viewer 显示预览
4. 切换到"深度设计"标签
5. 确认无"请先生成或加载一个基础设计"提示
6. 填写设计描述
7. 选择探索深度（快速/标准/深度）
8. 勾选优化策略
9. 点击"开始探索"
10. 等待完成 → 推荐方案卡片显示
11. 点击"设为当前方案"
12. 确认自动切换到"参数编辑"标签
13. 确认 ParameterPanel 加载了新版本参数

### W2: Variant 刷新

- 深度设计完成后，切换到参数编辑再切回深度设计
- 确认结果保留（推荐卡片、方案卡片、报告）

### W3: Timeline 渲染

- 运行中：确认显示中文阶段标签（解析设计目标、生成候选方案等）
- 运行阶段有 pulse 动画
- 已完成阶段显示耗时

### W4: Report 导出

- 点击"导出 .md"按钮
- 确认文件下载：`deep-design-{id}-{N}variants-{date}.md`
- 确认文件内容与页面报告一致

### W5: Markdown 渲染

- 确认报告中的标题、表格正确渲染
- 确认推荐标注正确显示

---

## SSE QA

### S1: 事件流完整性

运行快速探索（2 个方案），监控浏览器 Network 面板：

1. 确认 SSE 连接建立（`/api/deep-design/stream`）
2. 确认收到以下事件类型：
   - `graph_node`（阶段变更）
   - `variant_started`
   - `generation_complete`
   - `variant_completed`
3. 确认 SSE 连接在完成后正常关闭

### S2: 断线恢复

- 运行中刷新页面 → 确认面板回到空闲状态
- 重新运行 → 确认可以正常启动新的探索

## 错误恢复 QA

### E1: Variant 失败

- 如果某个 variant 生成失败：
  - 确认状态显示"✕ 失败"（红色）
  - 其他 variant 不受影响
  - 报告中正确反映失败数量

### E2: 重新运行

- 完成一次探索后
- 修改参数，再次点击"开始探索"
- 确认结果完全替换（不残留上一次数据）

### E3: 取消操作

- 运行中点击"取消"
- 确认 stream 停止
- 确认面板回到可编辑状态

# 深度设计探索 — 用户测试指南

本指南帮助你测试 AeroSpec 的深度设计探索功能。该功能基于 LangGraph 多代理工作流，自动生成多个飞机设计方案变体并进行比较和推荐。

---

## 1. 如何开始深度设计

1. **启动服务**（需要两个终端）：

   ```bash
   # 终端 1 — 后端
   set -a && . ./.env && set +a
   CAD_BACKEND=fake .venv/bin/python -m uvicorn services.api.app.main:app --host "$API_HOST" --port "$API_PORT"

   # 终端 2 — 前端
   cd apps/web
   set -a && . ../../.env && set +a
   npm run dev
   ```

2. **打开** http://localhost:3900

3. **生成初始方案**：在左侧 ChatPanel 对话框中描述你的设计需求，等待 AI 生成初始方案并加载 3D 模型。例如输入：

   > 设计一架翼展 12 米的双发无人机，巡航速度 200km/h，起飞重量 150kg

4. **切换到深度设计**：点击界面中的 **"深度设计"** 标签，进入 DeepDesignPanel。

> 如果未生成过初始方案，面板会提示"请先生成或加载一个基础设计"。

---

## 2. 推荐测试案例

以下三个场景覆盖了不同的优化方向：

| 场景 | 输入描述 | 优化策略建议 |
|------|----------|-------------|
| 航程优化 | `优化航程到 300km，保持起飞重量不变` | 长航时优化 |
| 长航时侦察 | `设计一架长航时侦察无人机，续航时间不低于 6 小时` | 长航时优化 + 载荷优化 |
| 短距起降 | `设计一架能在 200 米跑道起降的小型无人机` | 短距起降 |

---

## 3. 预期等待时间

| 后端模式 | 单方案耗时 | 3 方案总耗时 |
|----------|-----------|-------------|
| FakeCadBackend（默认） | 约 1-3 秒 | 约 3-10 秒 |
| OpenVSP 后端 | 约 30-60 秒 | 约 2-5 分钟 |

测试时建议使用 `CAD_BACKEND=fake`，可以快速迭代验证流程。

---

## 4. 探索深度选择

在"探索深度"选项中选择生成变体数量：

| 选项 | 变体数 | 适用场景 |
|------|--------|----------|
| 快速探索 | 2 | 快速验证，对比两个方案 |
| 标准探索 | 3 | 默认推荐，覆盖多种设计思路 |
| 深度探索 | 5 | 全面探索，生成更多变体对比 |

深度越高，等待时间越长，但方案多样性越好。

---

## 5. 探索目标策略

勾选一个或多个优化策略，AI 会根据策略调整参数方向：

- **长航时优化** — 侧重增大展弦比、降低巡航阻力、提高燃油效率
- **高速优化** — 侧重后掠翼、减小阻力、优化推重比
- **载荷优化** — 侧重增大机翼面积、提高结构承载能力
- **短距起降** — 侧重增升装置、降低翼载、优化失速特性

可以同时勾选多个策略，系统会在描述中自动拼接策略偏好。

---

## 6. 查看推荐方案

探索完成后，页面顶部会出现一张 **推荐方案卡片**（绿色高亮边框），显示：

- 推荐方案的名称和版本号（如 `v3`）
- AI 给出的推荐理由（从报告中自动提取）
- **"应用此方案"** 按钮

推荐逻辑基于报告中 `推荐：` 标记，若报告中无明确标记则默认推荐首个成功方案。

---

## 7. 查看方案差异

每个方案变体显示为独立卡片，包含：

- **方案名称** 和状态标签（succeeded / failed / running）
- **版本号**（如 `v1`、`v2`）
- **关键指标**（2 列网格布局）：
  - 翼展 (m)
  - 航程 (~km)
  - 升阻比 (L/D)
  - 展弦比
  - 翼载 (kg/m^2)

通过对比各方案的关键指标，可以直观了解不同设计方向的差异。

---

## 8. 设为当前方案

每个成功方案卡片底部有两个操作按钮：

- **"查看模型"** — 在 CAD 视图中加载该方案的 3D 模型，不切换参数面板
- **"设为当前方案"** — 加载 3D 模型 **并自动切换到参数编辑面板**，方便进一步微调

推荐方案卡片中的 **"应用此方案"** 按钮效果等同于"设为当前方案"。

---

## 9. 导出报告

探索完成后，页面底部显示 **"设计探索报告"**（Markdown 渲染），包含：

- 各变体的对比表格
- 关键参数差异分析
- 推荐方案说明

点击右上角 **"导出 .md"** 按钮，系统会下载一个 Markdown 文件，文件名格式为：

```
deep-design-{designId}-{变体数}variants-{日期}.md
```

## 10. 反馈问题

如果遇到问题，请通过 GitHub Issues 反馈：

- **仓库地址**：项目 GitHub 仓库的 Issues 页面
- **建议包含以下信息**：
  - 使用的是 FakeCadBackend 还是 OpenVSP
  - 输入的设计描述和选择的策略/深度
  - 期望行为与实际行为的差异
  - 浏览器控制台报错（F12 -> Console）
  - 后端终端日志

常见问题速查：

| 问题 | 解决方法 |
|------|----------|
| 页面样式/JS 404 | `rm -rf apps/web/.next && npm run dev` |
| 端口被占用 | `fuser -k 8900/tcp` 或 `fuser -k 3900/tcp` |
| 深度设计无反应 | 检查后端是否正常运行，确认 CAD_BACKEND 已设置 |
| 方案全部 failed | 检查 base spec 是否有效，尝试从对话中重新生成初始方案 |
